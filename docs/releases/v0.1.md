# Release v0.1 – First Meaningful Experience

Also referred to as the **Explorer Release** — specifically, the
external family-testing phase of this release. It is not a separate
release; design assets and documentation that mention "Explorer
Release" mean v0.1.

## Vision

Deliver the first complete DailyPaws experience.

A new family should be able to:

- Discover DailyPaws.
- Complete onboarding.
- Receive a personalised Mission.
- Complete the Mission.
- Reflect.
- Receive an Insight.
- Return the next day.

---

## Release Goals

- Validate the Product Model.
- Validate the Daily Mission loop.
- Validate the onboarding experience.
- Validate the tone of voice.

---

## Epics

### Epic 1 – Foundation

Cross-cutting technical and infrastructure work.

Features:

- Data Persistence (#16)
- Application Resilience and Error Handling (#25)
- PWA Installation (#26)
- Cross-cutting Technical Foundation

### Epic 2 – Onboarding

**Goal:** Help a new family start their DailyPaws journey in a way
that feels welcoming, calm and purposeful. By the end of onboarding,
DailyPaws should know enough to generate a meaningful first Mission
without overwhelming the family with questions.

Features:

- Welcome Experience (approved copy: `DesignDecisions.md` DD-001)
- Progressive Onboarding
- Dog Profile — Name, Life Stage (breed is deliberately not
  collected during onboarding, per PD-051 — learned later through
  Conversations, not asked upfront)
- Family Profile
- Relationship Goal (approved copy: `DesignDecisions.md` DD-002; the
  MVP supports one active Goal)
- Automatic Save
- Resume Onboarding
- Generate First Mission — every family receives the same shared
  first Mission regardless of Goal (approved copy:
  `DesignDecisions.md` DD-005); Goal-based personalisation begins
  with the second Mission

Guiding principle: the onboarding should collect only the
information required to generate a meaningful first Mission.
Everything else should be learned over time.

### Epic 3 – Home

**Goal:** Provide a calm daily starting point. Home should feel like
an invitation to today's experience rather than a dashboard.

Features:

- Today's Mission
- Mission Status
- Current Goal
- Timeline (lightweight MVP version)
- Simple Navigation

### Epic 4 – Mission

**Goal:** Help the family create meaningful everyday experiences
together.

Features:

- Mission Library
- Mission Selection
- Mission Personalisation
- Mission Details
- Mission Progress
- Mission Completion
- Next Mission Generation

Future idea, not v0.1: Optional Extra Missions.

### Epic 5 – Reflection

**Goal:** Help the family reflect on today's experience. Reflection
should feel quick and meaningful rather than like filling out a
form.

Features:

- Reflection Prompt
- Quick Responses
- Optional Comment
- Save Reflection
- Skip Reflection — if skipped, no new observations are created and
  no new Insight is generated from that Mission; the next Mission is
  generated using existing knowledge only

The shared first Mission (`DesignDecisions.md` DD-005) is the one
approved exception with its own Reflection options and Insight
mapping; no other Mission gets this treatment.

Note: "Reflection" is the internal product concept. The UI copy may
use more natural language, such as "How did it go?" Reflection is a
UX concept, not a Product Model concept — see `DesignDecisions.md`
DD-003.

### Epic 6 – Insights

**Goal:** Help the family develop a deeper understanding of their
dog over time.

Features:

- Insight Generation
- Evidence-based Insights
- Confidence-aware Language
- Insight Feedback
- Continuous Learning
- Insight History (lightweight MVP)

Do NOT expose Insight Categories in the v0.1 UI. `ProductModel.md`'s
four Insight Types (Dog, Relationship, Progress, Discovery) remain
an internal concept only — not user-facing in v0.1. No Product
Model change needed.

### Epic 7 – Conversations

**Goal:** Help DailyPaws continue learning through small purposeful
conversations.

Features:

- Follow-up Questions
- Conversation Context
- Structured Answers
- Continuous Learning
- Conversation Memory

Conversation in MVP means scripted or rule-based conversations.
AI-generated conversations remain out of scope.

Conversation Types (Discover, Reflect, Clarify, Celebrate, Coach)
are internal only, used by the Conversation Engine — same treatment
as Insight Types in Epic 6. Never exposed to the user; see
`DesignDecisions.md` DD-004.

### Epic 8 – Visual Identity

**Goal:** Bring DailyPaws' approved illustration style into the
product itself, so it carries the warm, calm visual identity
described in `docs/IllustrationGuide.md` rather than being text-only.

Features:

- Illustration System v1 (#62)

This Epic exists separately from Epic 1 (Foundation) and Epic 3
(Home) because the approved Illustration Guide and Illustration
System are a distinct product capability — cross-cutting across
Home, Mission and Insights, not Foundation- or Home-only work.

---

## Excluded

- Multiple dogs
- Multiple families
- Social
- Gamification
- Notifications
- Journey (full chaptered narrative — a simplified Timeline of
  recent activity on Home remains in scope, per MVP_PLAN.md's
  Timeline section)
- Moments
- AI Conversation — see Epic 7
- Optional Extra Missions — see Epic 4
- Insight Categories (not exposed in the UI — see Epic 6)
