# Implement Issue

When asked to implement a GitHub Issue:

1. Read DEVELOPMENT_PLAYBOOK.md.
2. Read the GitHub Issue.
3. Read all referenced documentation.
4. Implement only the approved scope.
5. Keep the implementation as simple as possible.
6. Validate:
   - lint
   - typecheck
   - build
   - required Playwright tests
7. Update documentation only if required by the Playbook.
8. Open a Pull Request.
9. Produce the standard implementation report.
10. Stop.

Never continue automatically to another issue.
