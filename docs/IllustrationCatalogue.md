# Purpose

The Illustration Catalog defines every illustration that exists (or is planned) in DailyPaws.

It is the single source of truth for the illustration library.

Each illustration has:

- a unique ID
- a clear purpose
- where it is used
- the emotion it should communicate
- a prompt
- a status

Illustrations should always follow the principles defined in **IllustrationGuide.md**.

---

# Status

Illustrations move through the following workflow:

- Planned
- Prompt Ready
- Generated
- Approved
- Implemented

---

# Versioning

Illustrations may have multiple generated versions before approval.

Example:

- I-001 Home Hero v1
- I-001 Home Hero v2
- I-001 Home Hero v3

Only one illustration becomes the approved production version.

Approved assets are stored in:

design/approved/

# Explorer Release Decisions

("Explorer Release" is not a separate product release — it's the
external family-testing phase of `docs/releases/v0.1.md` – First
Meaningful Experience.)

The Explorer Release intentionally uses:

- one illustration style
- one default adult woman
- one default medium-sized brown doodle

Illustration personalization is intentionally out of scope.
